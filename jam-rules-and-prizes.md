# Gamedev.js Jam 2026 — Rules, Prizes & Eligibility

> **Official site:** [gamedevjs.com/jam/2026](https://gamedevjs.com/jam/2026/)
> **Submit on:** [itch.io/jam/gamedevjs-2026](https://itch.io/jam/gamedevjs-2026)

## Key Facts

- **Dates:** April 13th, 5 PM CET – April 26th, 5 PM CET (13 days)
- **Theme:** [Machines](https://gamedevjs.com/competitions/gamedev-js-jam-2026-start-and-theme-announcement/) — interpret however you like, but judged on theme fit
- **Platform:** HTML5 game playable in a browser, no extra plugins
- **Submission:** Via [Itch.io](https://itch.io/jam/gamedevjs-2026) during the jam window
- **AI use:** Allowed and encouraged, as long as no copyright laws are broken
- **Content:** Must be **new content** created for the jam — no old demos or finished projects
- **Language:** English as default
- **Team size:** Solo or team, no hard limit; multiple entries allowed
- **Voting:** April 26 – May 9 (13 days), only participants who submitted can vote
- **Winners:** Overall announced May 10, challenge winners May 11
- **Voting criteria:** Innovation, Theme, Gameplay, Graphics, Audio (Overall = sum of all)
- **Community:** [#gamedevjs on X/Twitter](https://x.com/hashtag/gamedevjs) · [Discord](https://gamedevjs.com/discord)

## Rules

1. Must run in a web browser without plugins (HTML5)
2. New content created during the jam window
3. English as default language
4. Submitted via [Itch.io](https://itch.io/jam/gamedevjs-2026) between April 13–26
5. Any game engine or library is fine as long as the build is browser-playable

### Recommended Engines

[Phaser](https://phaser.io/) · [Defold](https://defold.com/) · [GDevelop](https://gdevelop.io/) · [Cocos](https://cocos.com/) · [8XR](https://engine.8xr.io/) · [Kaplay](https://kaplayjs.com/)

For multiplayer: [Colyseus](https://colyseus.io/) · [Modd.io](https://www.modd.io/)

Lightweight/constrained: [Kontra](https://straker.github.io/kontra/) · [Goodluck](https://gdlck.com/) · [LittleJS](https://killedbyapixel.github.io/LittleJS/docs/)

## Challenge Prizes & Prerequisites

Opt-in challenges with separate judging panels. You can enter multiple simultaneously.

### 1. Open Source Challenge — [GitHub](https://github.com/)

- **Prerequisite:** Publish your source code on GitHub in readable form
- **Prize:** 5× [GitHub Copilot Pro](https://github.com/features/copilot) (12 mo)
- **Judge:** Stacey Haffner (Microsoft OSPO Director)
- **Bonus:** Chance to be featured on the [GitHub Blog](https://github.blog/) (see [2022](https://github.blog/2022-05-31-top-games-source-code-from-gamedev-js-jam-2022/), [2023](https://github.blog/2023-06-21-gamedev-js-2023/), [2024](https://dev.to/github/gamedevjs-open-source-challenge-winners-5076))

### 2. Build it with Phaser — [Phaser Studio](https://phaser.io/)

- **Prerequisite:** Build your game using any version of [Phaser](https://phaser.io/)
- **Prizes:**
  - 10× [Phaser Editor Pro](https://phaser.io/editor) annual subscription
  - 5× [Vampire Survivors](https://store.steampowered.com/app/1794680/Vampire_Survivors/) bundle (game + 5 DLCs each, ~$5500 total)
- **Judge:** Richard Davey (Phaser creator) and team
- **Resources:** [Phaser docs](https://phaser.io/)

### 3. YouTube Playables — [YouTube](https://www.youtube.com/playables)

- **Prerequisites:**
  - **Single player** game
  - Not too violent, not specifically made for kids
  - No graphic/realistic violence, hate speech, sexual content, self-harm depiction, or IP infringement
  - Should entertain YouTube audiences
  - Review [YouTube community guidelines](https://www.youtube.com/howyoutubeworks/our-policies/)
  - Recommended: fill out [YouTube Playables Interest Form](https://docs.google.com/forms/d/e/1FAIpQLSdvdQ0lgIq2369aemj1O6w8R8FwGn9O5ARRGODDUbVINCRJQ/viewform), join [Playables Discord](https://discord.gg/eEFBtMswKT)
- **Prizes:**
  - 1× featured marketing in Discord
  - 7× [Gemini AI](https://gemini.google.com/) Pro (12 mo)
  - 10× fast track to Playables platform
  - 10× swag
- **Judges:** Lu Liu, Paul Bliudzius, Meg Campbell
- **Dev portal:** [developers.google.com/youtube/gaming/playables](https://developers.google.com/youtube/gaming/playables)

### 4. Deploy to Wavedash — [Wavedash](https://wavedash.com/)

- **Prerequisite:** Deploy and publish your game on [Wavedash](https://docs.wavedash.com/) (any engine)
- **Prizes:**
  - 1st place: **$1,000**
  - 2nd place: **$750**
  - 3rd place: **$500**
  - Honorable mention: **$250**
- **Judges:** Matt Portner, George Kennedy, Kyler Blue (Wavedash founders)
- **Docs:** [docs.wavedash.com](https://docs.wavedash.com/)
- **Support:** [Wavedash Discord](https://discord.gg/7HJDmfeRev)

### 5. Ethereum Challenge — [OP Guild](https://opgames.org/)

- **Prerequisite:** Use [Ethereum](https://ethereum.org/) ecosystem tech meaningfully (on-chain assets, wallets, smart contracts, etc.)
- **Prize:** 1st place: **$130**
- **Judge:** Paul Gadi (CTO/Co-Founder, OP Games)

## Relevance to Scrap Machine

| Challenge | Eligible? | Notes |
|-----------|-----------|-------|
| **Main jam** | Yes | Strong theme fit — "Machines" is our core concept |
| **Open Source (GitHub)** | Yes | Free to enter — just make the repo public and readable |
| **Build it with Phaser** | If we choose Phaser | Phaser is a jam sponsor; biggest challenge prize pool (~$5500) |
| **YouTube Playables** | **No** | Requires single player — our game is multiplayer (10–20 players) |
| **Deploy to Wavedash** | Yes | Engine-agnostic, biggest cash prizes ($2,500 total), just deploy there |
| **Ethereum** | Not practical | Would add unnecessary complexity for only $130 |

### Recommended Challenges to Target

1. **Deploy to Wavedash** — largest cash prize pool, minimal extra effort (just deploy)
2. **Open Source (GitHub)** — zero cost, just host code publicly in readable form
3. **Build it with Phaser** — if we commit to Phaser as our engine (aligns with jam sponsor)
